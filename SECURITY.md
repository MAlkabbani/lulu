# Security Policy

## Supported Scope

Lulu VAIA currently targets:

- macOS on Apple Silicon
- fully local Ollama-based chat and embedding workflows
- local memory persistence through ChromaDB

## Reporting A Security Issue

Please do not open public GitHub issues for suspected vulnerabilities.

Instead:

1. prepare a minimal reproduction or clear description
2. include impact, affected files, and any suggested mitigation
3. send the report through a private maintainer-owned contact channel

Until a dedicated security mailbox is published, use a private maintainer contact method rather than a public issue.

## Good Reports Include

- what is affected
- how the issue can be triggered
- whether local files, prompts, tool calls, or shell workflows are involved
- whether the issue can leak data, corrupt memory state, or execute unintended commands

## Disclosure Preference

Please allow time for investigation, mitigation, and coordinated disclosure before sharing details publicly.
